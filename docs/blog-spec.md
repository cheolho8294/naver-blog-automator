# 네이버 블로그 초안 체크리스트 (2026 운영 기준)

글 생성 시 아래를 만족해야 한다 (실제 상위노출을 보장하지는 않으며, 품질·일관성 목적이다).

## 제목

- 핵심 키워드는 제목 **앞쪽**에 둔다.
- 숫자·직접 경험·솔직함이 드러나는 패턴을 사용한다.
- 허위 체험·과장 클릭베이트는 금지.

## 톤·가독성 (모바일)

- **존댓말·친접한 설명체** (~습니다/~해요). 반말·명령형 지양.
- 휴대폰 화면 기준으로 **짧은 호흡**. 한 단락(빈 줄 구분)은 대략 **4~6줄** 이내가 되게 줄바꿈과 공백을 넣는다.
- **D.I.A**: 도입에서 관심·공감·제목 약속 반영 → 본문에서 정보·비교·FAQ → 마무리에서 행동(댓글·저장·다음 확인).
- **D.I.A+ 관점**(검색 의도 정렬 참고 표현 — 노출 보장 없음): 주제·키워드를 검색 진입처로 간주하고, 정보형·경험형·선택형에 맞춰 순서와 분량을 맞춤. 한 글 안 완결, 제목과 본문 일치. 동의어는 자연스러울 때만 아주 적게.

## 검색 의도·품질

- 글 하나로 독자의 **대표 질문**에 빠진 답이 없도록 FAQ·본문 분담한다.
- **초행 따라 함** 가능한 단계와 주의(메모·사진 근거 범위)를 제시한다. 불확실하면 과장하지 않고 범위를 밝힌다.
- **honestDownsides**(아쉬운 점 3가지)로 신뢰와 의도 충족의 균형을 맞춘다.

## 본문 구조

- 소제목은 `소제목1`, `소제목2`처럼 번호를 붙여 구획한다.
- 최소 **5개** 이상의 소제목 섹션.
- 한 단락에는 핵심 **한 가지**만 담는다.
- 독자가 다음 단락을 읽어야 결론이 보이는 흐름을 만든다.
- 표는 **사용자 메모·사진·조사 스니펫에 근거**가 있을 때만 넣는다. 근거 없는 숫자를 지어내지 않는다.
- 사진 마커 `[Gn]` 다음에는 한 번 띄우고 설명 글을 이어간다.

## 경험·비교 (작업 후기)

- **사진·사용자 설명**을 중심으로 작업 후기를 쓴다. 현장에서 흔한 오염·스월·미세 스크래치는 자연스럽게 언급 가능.
- **과도한 추측 금지**: 깊은 스크래치, 재도장, PPF 손상, 코팅 박리, 사고 흔적, 알칸타라/카본/무광 PPF → 사진·메모 있을 때만.
- **결과 표현**: 완벽 제거·신차급·완전 복원 대신 눈에 띄게 개선·광택 회복·스월 감소·반사감 선명 등.
- **현장 디테일러가 고객에게 설명하듯** 자연스럽게. 작업 흐름: 입고·상태확인 → 세차·전처리 → 메인 → 마무리 → 최종 결과.
- **관찰·경험 우선**: 「일반적으로/보통/주로/대부분」 백과사전식 최소, 이번 차량 **관찰 내용** 중심. 공정마다 **왜 했는지** 포함.
- **차량 특징**: 사진·메모로 확인 가능한 형상·부위(넓은 측면, 대형 전면, 슬라이딩 도어 등) 본문 반영.
- **현장 체감 1~2문장** 필수(구체적 관찰·이유, 막연한 감탄 금지).
- **FAQ**: 해당 작업 유형(광택·코팅·실내·에바·가죽 등) **고객 상담형** 질문 우선.
- **수치**(시간, RPM, 희석, 코팅 지속, 관리 주기)는 실제 정보 있을 때만.
- 최종 기준: **현장에서 고객에게 자연스럽게 말할 수 있는 문장**인가.
- **아쉬웠던 점 3가지**를 솔직하게 별도 블록으로 제시한다.
- **이전 vs 이번** 비교를 포함한다.

## 분량·형식

- 공백 제외 **1,200자 이상**(마커·HTML 제외, tag·제목·본문 등 합산).
- 도입 요약 약 **120~180자**.
- FAQ **2~8문항**.
- 말미에 댓글 유도·저장 권유·관련 글 자리를 넣는다.

## 이미지 계획

- `imagePlan`은 업로드된 원본 인덱스와 일치해야 한다.
- 본문에는 필요한 위치에 `[G1]` 형태 그룹 마커를 넣는다.
- `imagePlan.ordered[].section`은 글머리 `sections` 배열의 **0-based** 인덱스다(예: 소제목1 블록=0).
- 각 **groupId**당 사진 수는 **2장 또는 4장**(네이버 앱 2열·2×2 콜라주).
- 업로드가 **8장 이상**이면 본문 사용 장수는 **8~13장 중 짝수**만 가능 (2·4장 묶음의 합). 나머지는 `excluded`에 사유.

## 조사 자료

- 서버에 `TAVILY_API_KEY`가 있으면 검색 스니펫이 주입될 수 있다. 인용은 스니펫 범위 안에서만 한다.

## 배포

- 생성된 HTML은 네이버 스마트에디터에 붙여 넣는다. 글쓰기 진입은 앱/웹의 **글쓰기** 메뉴를 사용한다.
